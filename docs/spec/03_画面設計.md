# 03. 画面設計

## ルーティング

ベースパス: `/contents/sm`

| パス | 画面 | コンポーネント |
|---|---|---|
| `/` | ランディングページ | `LandingView` |
| `/app` | ダッシュボード（最近のノート） | `DashboardView` |

## 画面構成

### ランディングページ (`LandingView`)

- プロダクト紹介とCTA
- `/app` への導線

### ダッシュボード (`DashboardView`)

**レイアウト:**
- デスクトップ: 左サイドバー（ナビゲーション）＋メインコンテンツ
- モバイル: メインコンテンツ＋画面下部ボトムナビゲーション

**ナビゲーション項目:**
- 最近のノート
- お気に入り
- ノート一覧
- タグ
- カテゴリ（エクスプローラーへ遷移）

**メインコンテンツ:**
- `NoteGrid`: ノートカードのグリッド表示
- `NoteCard`: サムネイル画像＋タイトル＋更新日

### カテゴリエクスプローラー (`CategoryExplorerView`)

**レイアウト:** 2ペイン構成
- **左ペイン:** カテゴリツリー（`CategoryTreeNode` を再帰的に描画）
- **右ペイン:** 選択カテゴリのサブカテゴリグリッド＋ノートカード

**操作:**
- ツリーノードのクリックで展開・折りたたみ
- カテゴリの作成・名前変更・削除（モーダルダイアログ）

### ノート閲覧 (`NoteView`)

**ヘッダー:** タイトル・タグ・パンくずリスト・「編集」「ヒント編集」ボタン

**工程リスト:** `StepCard` を順番に表示
- 工程番号・タイトル
- 写真（トリミング範囲を考慮した表示）
- 説明文・ハイライトブロック
- ヒント欄（ヒント編集モード時のみ編集可能）

### ノート編集 (`NoteEditView`)

**ヘッダー:** タイトル入力・カテゴリ選択・タグ入力（`TagInput`）

**工程リスト:**
- `StepCard`: ドラッグハンドル付き工程カード
- `StepInsertButton`: 工程間の＋ボタン
- 各工程に写真追加メニュー（`PhotoAddMenu`）

**フッター:** 保存ボタン・閲覧モードへ戻るボタン・保存確認メッセージ

## StepCardレイアウト

全画面サイズ共通のレイアウト:

```
┌──────────────────────────────────┐
│ タイトル                           │
├───────────────┬──────────────────┤
│               │ 説明              │
│   画像        │                  │
│               │ ヒント            │
├───────────────┴──────────────────┤
│ その他コントロール（ハイライト追加等）│
└──────────────────────────────────┘
```

- 上段: 工程番号＋タイトル
- 中段: 左右2列（左=画像、右=説明＋ヒント）
- 下段: ハイライトブロック追加ボタン等のコントロール
- 画像クリックでライトボックス（全画面表示）
- 編集モード時、各画像の右上に×（削除）ボタンを表示

## コンポーネント

| コンポーネント | 役割 |
|---|---|
| `StepCard` | 工程カード。写真・説明・ハイライト・ヒントを表示 |
| `StepInsertButton` | 工程間の境界線中央に＋ボタンを表示 |
| `PhotoThumbnail` | トリミング範囲を考慮した画像サムネイル |
| `PhotoAddMenu` | 写真追加メニュー（撮影・アップロード・クリップボード） |
| `HighlightBlock` | 警告・ポイント・メモの強調ブロック |
| `AddHighlightButton` | ハイライトブロック追加ボタン群 |
| `TagInput` | タグ入力＋サジェスト |

## レスポンシブ対応

| ブレークポイント | 対象 |
|---|---|
| `sm` (640px) | 小型タブレット |
| `md` (768px) | タブレット |
| `lg` (1024px) | デスクトップ |

- ダッシュボードのナビゲーション: lg以上でサイドバー、lg未満でボトムナビ
- ノートカードのグリッド: 画面幅に応じてカラム数を調整

## モバイルUI設計ルール

`docs/モバイルUIガイドライン.md` に基づく実装ルール:

### フォーム要素
- `input`, `textarea`, `select`, `button` は `text-base` (16px) 以上
- iOS Safari のフォーカス時自動ズームを防止するため

### タップ領域
- タップ可能要素は 48×48px 以上の操作領域を確保
- アイコンボタンは `p-3` (12px padding) 以上
- 小さい要素同士は 8px 以上離す

### 画面高さ
- `min-h-screen` (100vh) ではなく `min-h-dvh` (100dvh) を使用
- モバイルブラウザのアドレスバーによる表示崩れを防止

### safe-area
- 下部固定UIは `pb-[env(safe-area-inset-bottom)]` を考慮
- iPhone のノッチ・ホームバーとの重なりを防止

### テキストサイズ
- 本文: 16px以上
- 補助テキスト: 14px (`text-sm`) 以上
- `text-xs` (12px) は日付・メタデータ等の非対話要素のみ許容

## カラーシステム

### 設計思想

対象ユーザー（製造現場の作業者）に配慮した配色:
- 眩しくない背景色（`oklch(0.88 0 0)` 付近のダークトーン）
- 集中しやすく気が散らないデザイン
- 説明文の読みやすさを重視
- ノート内容の背景はやや明るめの色

### カラーパレット

- プライマリ: Blue-600系
- フォルダアイコン: Amber-400系
- 危険操作: Red系
- アイコン: lucide-react
