## アーキテクチャ

### MVVM + Repository パターン

```
View (React Component)
  ↓ props / hooks
ViewModel (React Custom Hook)
  ↓ メソッド呼び出し
Repository (Interface → API実装)
  ↓ HTTP
API (PHP)
  ↓ StorageInterface
Storage (SQLite or Filesystem)
```

### ViewModel層

ViewModelはReact Custom Hookとして実装。UIロジックとデータ操作を担当:

| ViewModel | 責務 |
|---|---|
| `useNoteEditorViewModel` | ノート編集状態管理（draft, isDirty, 工程CRUD, ハイライト操作） |
| `useCategoryViewModel` | カテゴリ一覧・子カテゴリ絞り込み・パンくずリスト生成 |
| `useDashboardViewModel` | ダッシュボードのアクティブセクション判定 |
| `useCategoryTreeViewModel` | ツリーの展開・折りたたみ状態管理 |

### Repository層

`repositories/types.ts` でインターフェースを定義し、`ApiNoteRepository.ts` で実装。シングルトンとしてエクスポート:

```typescript
export const noteRepo = new ApiNoteRepository();
export const categoryRepo = new ApiCategoryRepository();
export const tagRepo = new ApiTagRepository();
```

### ディレクトリ構成

```
frontend/src/
  models/        ← 型定義
  repositories/  ← APIアクセス層（インターフェース＋実装）
  viewmodels/    ← ViewModel（React Hooks）
    __tests__/   ← ViewModelのテスト
  views/         ← ページレベルコンポーネント
  components/    ← 再利用可能UIコンポーネント
  hooks/         ← 汎用カスタムHooks
  utils/         ← ユーティリティ関数
  test/          ← テストセットアップ
```

## PWA対応

`vite-plugin-pwa` による構成:

- **登録方式:** autoUpdate（自動更新）
- **スコープ:** `/contents/sm/`
- **開始URL:** `/contents/sm/app`
- **表示モード:** standalone
- **キャッシュ対象:** `*.js`, `*.css`, `*.html`, `*.svg`
- **API除外:** `/contents/sm/api` 以下はキャッシュしない（`navigateFallbackDenylist`）
- **ナビゲーションフォールバック:** `/contents/sm/index.html`

## リソース隔離

`door-fujita.com` ドメインで複数アプリが同居するため、以下のルールを適用:

| カテゴリ | プレフィックス | 例 |
|---|---|---|
| Cookie | `sm_` | `sm_session`（`path=/contents/sm/`） |
| LocalStorage | `sm_` | `sm_data` |
| DBテーブル | `sm_` | `sm_notes`, `sm_categories`, `sm_tags` |

Viteの `base` オプション: `/contents/sm/`

## デプロイ

| 環境 | URL |
|---|---|
| 開発フロント | `http://localhost:5179/contents/sm/` |
| 開発API | `http://localhost:8004/api`（Viteプロキシ経由） |
| 本番フロント | `https://door-fujita.com/contents/sm/` |
| 本番API | `https://door-fujita.com/contents/sm/api/` |

### APIルーティング（.htaccess）

- `api/data/` と `api/storage/` への直接アクセスを403で遮断
- ファイル・ディレクトリ以外のリクエストを `index.php` にルーティング
- `uploads/` ディレクトリは静的ファイルとして直接配信

### 開発サーバー

Viteの開発サーバーが `/contents/sm/api` へのリクエストを `http://localhost:8004/api` にプロキシ。
