# 06. 変更履歴

## 2026-03-24: 写真ドラッグ&ドロップ機能の仕様追加

**新機能仕様:**
- 未割り当て写真を工程カードへドラッグ&ドロップで割り当て
- 工程内写真の長押しドラッグで順番入れ替え
- 工程間の写真移動（ドラッグで別工程へ）
- 工程写真を未割り当てプールへドラッグで戻す
- 編集モードで未割り当てプールを常時表示（空でもドロップ先として表示）
- ドロップ先ハイライト表示

**技術方針:** @dnd-kit（既存の工程並び替えと共存）、PointerSensor + TouchSensor

**理由:** 現場での写真整理を直感的に行えるようにする。右クリックメニューだけでは操作が煩雑

**新機能仕様:**

## 2026-03-22: 既存画像サムネイル一括生成

**新機能:**
- サーバー上でPHP GDを使った既存画像のサムネイル一括生成スクリプト追加
- 生成後、ノートJSONデータ内のphotoにthumbnailUrlを自動付与

**理由:** サムネイル導入前にアップロードされた画像がオリジナルサイズで表示されダッシュボードが重い

## 2026-03-22: 動画対応・サムネイル・設定画面・アノテーションバグ修正

**新機能:**
- 動画アップロード対応（MP4/WebM/MOV）
- iPhone風動画トリミングUI（FFmpeg.wasmによるブラウザ内処理）
- 画像サムネイル自動生成（アップロード時にCanvas APIで長辺480pxに縮小）
- アプリ設定画面（/settings）新設 — 動画出力形式・最大長さ・サムネイルサイズ

**バグ修正:**
- react-image-editor-lightbox のアノテーション初期位置がキャンバス外に配置される問題を修正（baseImage.widthではなくcanvasLogicalWを使用）

**データモデル変更:**
- Photo型に `mediaType`, `duration` フィールドを追加
- API uploadPhotoレスポンスに `thumbnailUrl`, `mediaType` を追加

## 2026-03-21: モバイルUIガイドライン適用

**修正内容:**

- 写真削除ボタン・工程挿入ボタンのタップ領域を拡大

**バグ修正:**

## 2026-03-21: カテゴリ・編集UX・レイアウト改善

**カテゴリ管理:**
- 「未分類」デフォルトカテゴリを追加（ID固定・削除/名前変更不可・ツリー最下部）
- カテゴリ閲覧中のノート作成でそのカテゴリを初期設定

**ノート編集:**
- 各画像右上に×（削除）ボタンを追加
- ヘッダーにUndo（元に戻す）ボタンを追加

**StepCardレイアウト:**
- 全画面サイズ共通で上=タイトル、中=左画像+右説明ヒント、下=コントロールに変更

**画像編集:**
- 初期表示を画面90%に収めるよう制御
- ピンチイン・ピンチアウトで拡大縮小を追加

**理由:** 現場での操作性向上。間違いを恐れず作業できるUX、スマホ・タブレット共通のレイアウト統一。
