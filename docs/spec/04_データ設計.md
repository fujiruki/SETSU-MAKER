## SQLiteスキーマ

```sql
CREATE TABLE sm_categories (
    id TEXT PRIMARY KEY,
    name TEXT NOT NULL,
    parent_id TEXT,
    sort_order INTEGER DEFAULT 0,
    created_at TEXT NOT NULL
);

CREATE TABLE sm_tags (
    id TEXT PRIMARY KEY,
    name TEXT NOT NULL UNIQUE,
    created_at TEXT NOT NULL
);

CREATE TABLE sm_notes (
    id TEXT PRIMARY KEY,
    title TEXT NOT NULL,
    category_id TEXT NOT NULL,
    steps_json TEXT NOT NULL DEFAULT '[]',
    tag_ids_json TEXT NOT NULL DEFAULT '[]',
    eyecatch_photo_id TEXT,
    handwriting_data TEXT,
    is_favorite INTEGER DEFAULT 0,
    created_at TEXT NOT NULL,
    updated_at TEXT NOT NULL,
    unassigned_photos_json TEXT NOT NULL DEFAULT '[]'
);
```

- `steps_json`: Step配列をJSON文字列として格納
- `tag_ids_json`: タグID配列をJSON文字列として格納
- `unassigned_photos_json`: 未割り当て写真のPhoto配列をJSON文字列として格納
- テーブル名は `sm_` プレフィックスで名前空間を分離

## ストレージモード

`api/config.php` の `STORAGE_MODE` で切替:

| モード | 保存先 | 用途 |
|---|---|---|
| `sqlite` | `api/database.sqlite` | 本番推奨。PDO + WALモード |
| `filesystem` | `api/data/notes/{noteId}/note.json` | ファイル管理で内容を直接確認したい場合 |

### ファイルシステムモードの構造

```
api/data/
  categories.json
  tags.json
  notes/
    {noteId}/
      note.json   ← ノートの全データ（JSON）
      *.jpg/png   ← アップロード画像（co-location）
```

### 画像保存

両モード共通で `api/uploads/{noteId}/` に保存。ファイルシステムモードでは `api/data/notes/{noteId}/` にもコピーして画像とノートデータを同一フォルダに配置。

ファイル名形式: `{YYYYMMDD_HHmmss}_{hash6}.{ext}`（EXIF撮影日時優先）

## APIエンドポイント

ベースURL:
- 開発: `http://localhost:8004/api`
- 本番: `https://door-fujita.com/contents/sm/api`

フロントエンドからは `POST` + `X-HTTP-Method-Override` ヘッダーで `PUT`/`DELETE` を送信。

### ヘルスチェック

| メソッド | パス | レスポンス |
|---|---|---|
| GET | `/health` | `{status: "ok", storage: "sqlite"}` |

### カテゴリ

| メソッド | パス | リクエスト | レスポンス |
|---|---|---|---|
| GET | `/categories` | — | `Category[]` |
| POST | `/categories` | `{name, parentId?, order?}` | `Category` |
| PUT | `/categories/{id}` | `{name?, parentId?}` | `Category`（`uncategorized`は拒否） |
| DELETE | `/categories/{id}` | — | `{ok: boolean}`（`uncategorized`は拒否） |

**デフォルトカテゴリ「未分類」:**
- ID: `uncategorized`（固定値）
- バックエンド初回起動時に自動作成
- PUT/DELETEリクエストは400エラーを返す

### タグ

| メソッド | パス | リクエスト | レスポンス |
|---|---|---|---|
| GET | `/tags` | クエリ: `?q={query}` | `Tag[]`（最大20件） |
| POST | `/tags` | `{name}` | `Tag`（既存なら既存を返却） |
| DELETE | `/tags/{id}` | — | — |

### ノート

| メソッド | パス | リクエスト | レスポンス |
|---|---|---|---|
| GET | `/notes` | クエリ: `?categoryId=&favorite=1&q=&limit=` | `NoteListItem[]`（最大100件） |
| POST | `/notes` | `{title, categoryId}` | `Note` |
| GET | `/notes/{id}` | — | `Note` |
| PUT | `/notes/{id}` | `Note`オブジェクト | `Note` |
| DELETE | `/notes/{id}` | — | `{ok: boolean}` |

### 画像アップロード

| メソッド | パス | リクエスト | レスポンス |
|---|---|---|---|
| POST | `/notes/{noteId}/steps/{stepId}/photos` | `FormData` (`photo`フィールド) | `{url, filename, takenAt}` |
| POST | `/notes/{noteId}/photos` | `FormData` (`photo`フィールド) | `{url, filename, takenAt}`（未割り当て写真） |

対応ファイル形式: `jpg`, `jpeg`, `png`, `webp`, `gif`

## Repositoryインターフェース

フロントエンドはRepositoryパターンでAPIアクセスを抽象化:

```typescript
interface NoteRepository {
  getList(categoryId?: string, tagId?: string): Promise<NoteListItem[]>;
  getById(id: string): Promise<Note | null>;
  getRecent(limit?: number): Promise<NoteListItem[]>;
  getFavorites(): Promise<NoteListItem[]>;
  search(query: string): Promise<NoteListItem[]>;
  create(categoryId: string, title: string): Promise<Note>;
  update(note: Note): Promise<Note>;
  delete(id: string): Promise<void>;
  toggleFavorite(id: string): Promise<void>;
  uploadPhoto(noteId: string, stepId: string, file: File): Promise<string>;
}

interface CategoryRepository {
  getAll(): Promise<Category[]>;
  getById(id: string): Promise<Category | null>;
  create(name: string, parentId: string | null): Promise<Category>;
  update(id: string, patch: { name?: string; parentId?: string | null }): Promise<Category>;
  delete(id: string): Promise<void>;
}

interface TagRepository {
  getAll(): Promise<Tag[]>;
  create(name: string): Promise<Tag>;
  delete(id: string): Promise<void>;
  suggest(query: string): Promise<Tag[]>;
}
```
