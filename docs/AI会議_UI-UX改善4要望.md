# AI専門家会議: SETSU-MAKER UI/UX改善 4つの要望

## 1. 会議の定義
- **テーマ**: requests.mdの4つのUI/UX要望を、使いやすく初めてでもわかりやすいデザインにするための設計
- **ゴール**: 各要望に対する具体的なUI設計方針を決定
- **前提条件**: 施工現場の建具職人がiPhoneで使用、Tailwind CSS + lucide-react踏襲

## 2. キャスティング
1. 技術・実装担当（React/Vite実現可能性）
2. UX・ユーザー視点担当（現場職人の使い勝手）
3. ビジネス・マネタイズ担当（発信コンテンツ化の価値）
4. リスク・法・倫理担当（プライバシー・誤操作防止）

## 3. 結論

### 要望1: テキストエリア自動拡張 → Yes
- `scrollHeight` ベースで内容に合わせて高さ自動拡張
- 上限 `max-height: 50vh`、超過後はtextarea内スクロール
- 閲覧モードは `<p>` タグで全文表示（textarea不使用）
- 初期表示時にも内容量に合わせた高さ

### 要望2: 動画サムネイル → Yes
- アップロード時にCanvasで先頭フレームをキャプチャ → JPEG化 → サムネイルとしてアップロード
- 表示: `<img>` サムネイル + 暗いオーバーレイ(30%) + 再生ボタン
- 既存の画像サムネイルパイプライン（resizeImage + API thumbnailフィールド）を流用

### 要望3: 保存プログレス → Yes
- ヘッダー直下に2pxの薄いプログレスバー（sticky固定）
- 青色で進行、完了後に緑に変わりフェードアウト
- アップロード中の `beforeunload` 警告を追加
- 個々のファイルアップロード進捗を積算

### 要望4: note/SnapBaton投稿 → 条件付きYes
- Phase 1: 「共有用テキスト生成」（テキスト整形 + クリップボードコピー + 写真ダウンロード）
- Phase 2: SnapBaton API連携（自社のため直接投稿可能）
- note直接投稿はAPI制約のため保留
- 投稿前に「公開内容の確認」ステップ必須（プライバシーリスク対策）

## 4. 想定リスクと対策
- 動画サムネイル生成でiOS Safari問題 → `loadeddata` イベントで確実にフレーム描画
- 外部投稿時の顧客プライバシー → 確認ステップ必須
- テキストエリア無制限拡張 → max-height上限で制御
